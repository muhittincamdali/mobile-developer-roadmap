## Adding Resources

When adding a resource, please include:

```markdown
- [Resource Name](URL) - Brief description of what it covers
```

**Quality criteria:**
- The resource must be high-quality and well-maintained
- Free resources are preferred, but paid ones are welcome in the paid section
- The resource should be relevant to mobile development
- No affiliate links
- No self-promotion without disclosure

## Pull Request Process

1. **Fork** the repository
2. **Create** a feature branch (`git checkout -b feature/add-new-resource`)
3. **Make** your changes
4. **Test** all links you've added
5. **Commit** with a clear message
6. **Push** to your fork
7. **Open** a Pull Request

### PR Checklist

- [ ] All links are working
- [ ] Content follows existing format
- [ ] No duplicate entries
- [ ] Spelling and grammar checked
- [ ] Commit message follows conventions

## Style Guide

### Markdown

- Use `#` for main headings, `##` for sections, `###` for subsections
- Use `-` for unordered lists
- Use `1.` for ordered lists
- Add blank lines between sections
- Use code blocks with language identifiers

### Content

- Write in clear, concise English
- Use active voice
- Be specific about skill levels (beginner, intermediate, advanced)
- Include estimated time when possible
- Keep descriptions brief but informative

### File Organization

```
roadmaps/       ‚Üí Detailed roadmaps per platform
resources/      ‚Üí Learning resources (free, paid, YouTube)
projects/       ‚Üí Project ideas by difficulty
```

## Commit Messages

We follow [Conventional Commits](https://www.conventionalcommits.org/):

```
feat: add new iOS testing resources
fix: update broken link in flutter roadmap
docs: improve contributing guidelines
style: fix markdown formatting in README
```

### Types

| Type | Description |
|------|-------------|
| `feat` | New content or section |
| `fix` | Fix broken links, typos, incorrect info |
| `docs` | Documentation improvements |
| `style` | Formatting changes |
| `refactor` | Restructuring content |
| `chore` | Maintenance tasks |
