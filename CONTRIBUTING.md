## How to Contribute

### Suggesting Changes

1. Open an issue describing your suggestion
2. Explain why the change improves the roadmap
3. Include resources if adding new topics

### Types of Contributions

- **Add new topics**: Technologies or skills that should be included
- **Update resources**: Better or updated learning materials
- **Fix errors**: Correct outdated information
- **Improve structure**: Better organization or flow

### Submitting Changes

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/add-flutter-path`)
3. Make your changes
4. Commit (`git commit -m 'Add: Flutter learning path'`)
5. Push (`git push origin feature/add-flutter-path`)
6. Open a Pull Request

## Content Guidelines

### Roadmap Topics

- Must be relevant to mobile development
- Should be industry-recognized
- Include difficulty level
- Provide learning resources

### Resources

- Free resources preferred
- Include official documentation
- Avoid paywalled content when possible
- Verify links are working
